# دفتر مدير الفرع — تطبيق PWA لمحلات الذهب والفضة

تطبيق محاسبة كامل يعمل **دون اتصال** ومحفوظ بالكامل على جهازك فقط. مبني بـ HTML/CSS/JS خالص بدون أي أدوات بناء.

## الميزات الكاملة

- ✅ **الصفحة الرئيسية** — لوحة تحكم مع KPIs اليوم، بلاطات سريعة لكل دفتر، وتصدير/استيراد JSON للنسخ الاحتياطي.
- ✅ **دفتر اليومية** — قيود تفصيلية للذهب (18/21/22/24 × مشغول/كسر/صافي)، الفضة (925/999)، والنقد. يحسب الإجمالي تلقائياً، يدعم نقد/بنك/آجل، عرض تفصيلي وإجمالي، تنقل بالأيام، وطباعة.
- ✅ **دفتر الموردين** — قائمة موردين مع رصيد افتتاحي، حركات شراء/سداد، كشف حساب لكل مورد، إجمالي المستحق.
- ✅ **دفتر العملاء** — مماثل للموردين مع تركيز على البيع الآجل.
- ✅ **دفتر البنك والصندوق** — حسابات متعددة (بنكية + صندوق نقدي)، إيداع/سحب/تحويل بين الحسابات، رصيد جاري لكل حساب.
- ✅ **دفتر المصاريف** — تصنيفات (إيجار، رواتب، كهرباء…)، تقارير شهرية مع نسب التصنيفات.
- ✅ **دفتر السلف** — للموظفين والعملاء، مع جدول دفعات سداد لكل سلفة.
- ✅ **دفتر المخزون** — جرد القطع بالـ SKU، الوزن، العيار، الموقع، الحالة (متوفر/محجوز/مباع).
- ✅ **التقارير الشاملة** — تقارير شهرية وسنوية، حساب الربح الصافي، تفصيل التدفق النقدي، صورة عامة لكل الدفاتر.
- ✅ **الإعدادات + الفواتير** — بيانات المحل (اسم، فرع، عنوان، رقم ضريبي)، إصدار فواتير برقم تسلسلي تلقائي وطباعتها.
- ✅ **PWA كامل** — manifest + service worker + أيقونات. يثبَّت كتطبيق ويعمل offline بعد أول زيارة.

## التشغيل

التطبيق PWA ويجب تقديمه من خادم HTTP (لأن الـ Service Worker لا يعمل مع `file://`).

### الطريقة الأسرع (Python)

```bash
cd gold-ledger
python3 -m http.server 8080
```

ثم افتح: `http://localhost:8080`

### بدائل

- **Node.js:** `npx serve .`
- **VS Code:** امتداد *Live Server*
- **نشر على الإنترنت:** أي استضافة ثابتة (Netlify, Vercel, GitHub Pages، أو أي استضافة عادية).

> بعد أول زيارة أونلاين، سيعمل التطبيق بالكامل offline.

## تثبيت التطبيق على الهاتف/الكمبيوتر

- **أندرويد/كروم:** اضغط على "تثبيت التطبيق" في شريط العنوان أو من قائمة المتصفح.
- **iOS/سفاري:** زر المشاركة ← "إضافة إلى الشاشة الرئيسية".
- **ديسكتوب:** أيقونة التثبيت في شريط العنوان.

## البيانات والنسخ الاحتياطي

- جميع البيانات محفوظة في `localStorage` على جهازك فقط. لا يتم إرسالها إلى أي خادم.
- **مهم:** احرص على تصدير نسخة احتياطية بشكل دوري من الصفحة الرئيسية.
- استيراد النسخة الاحتياطية متاح على نفس الجهاز أو على جهاز آخر.

## التمديد

إضافة دفتر جديد تتطلب:
1. إنشاء ملف `js/pages/xxx.js` يُصدِّر دالة `renderXxx(container)`.
2. إضافة مفتاح للمخزن في `db.js` إذا احتاج تخزين جديد، عبر `makeStore(keys.xxx)`.
3. تسجيل المسار في `routes` داخل `js/app.js`.
4. إضافة الرابط في الشريط الجانبي في `index.html` وبلاطة في `home.js`.
5. تحديث قائمة `CORE_ASSETS` في `sw.js` ليشمل الملف الجديد.

الهيكل قابل للتوسع مباشرة.
